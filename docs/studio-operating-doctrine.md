# Studio operating doctrine — Cursor Game Studios

> **Work in progress.** This is the philosophical spine: how we behave when
> models, tools, and deadlines disagree.

## Non-negotiables

| Doctrine | Meaning |
|----------|---------|
| **Human EP** | The human is executive producer. Agents advise; they do not usurp. |
| **Evidence over aura** | A claim without a diff, test, or doc citation is gossip. |
| **Single writer queue** | Parallel research; serialized mutation on shared surfaces (especially engine/editor MCP). |
| **Small batches** | Prefer reviewable slices over heroic rewrites. |
| **Boring Git** | Linear or trunk-based discipline beats romance branching. |

## The Cursor-native premise

Cursor gives you **rules**, **skills**, and **subagents**. This repository treats
them like a **studio org chart** encoded in files: predictable routing, explicit
handoffs, and zero dependency on a single chat monologue.

## What “power” means here

Power is not verbosity. Power is **repeatable outcomes**:

- A skill that can be run Monday and Thursday with the same gates.
- An agent that refuses ambiguous scope without a decision.
- A matrix that tells you which specialist to load before you burn context.

## Graveyard humor clause

If the coffee is bad and the build is red, we still behave like professionals:
we log, we bisect, we fix. The old tools can rest. We have work to do.
